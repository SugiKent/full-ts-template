# 外部サービスドキュメント

このディレクトリには、システムで利用する外部サービス（API、SDK、SaaS等）の仕様ドキュメントを格納します。

## 目的

- AI エージェントが実装時に正確な外部サービス仕様を参照できるようにする
- 開発者が外部サービスの使用方法を素早く把握できるようにする
- プロジェクト固有の実装ガイダンスを提供する

## ドキュメント作成規約

### ファイル命名規則

```
{SERVICE_NAME}_API.md
```

例：
- `STRIPE_API.md`
- `SENDGRID_API.md`
- `OPENAI_API.md`

### 必須セクション

各ドキュメントには以下のセクションを含めること：

1. **概要** - 用途、公式ドキュメントURL、使用バージョン
2. **認証** - 認証方式、環境変数名
3. **主要エンドポイント/機能** - このプロジェクトで使用する機能
4. **制約事項** - レート制限、利用上の注意
5. **エラーハンドリング** - 主要なエラーコードと対処法
6. **実装時の注意点** - プロジェクト固有のガイダンス

### 禁止事項

- APIキー、シークレット、認証情報を記載しない
- 本番環境の具体的なエンドポイントURLを記載しない
- 個人情報を含むサンプルデータを使用しない
