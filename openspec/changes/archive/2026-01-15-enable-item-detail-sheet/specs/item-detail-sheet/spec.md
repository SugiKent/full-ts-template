# item-detail-sheet Specification Delta

## ADDED Requirements

### Requirement: アイテム詳細シートモーダル（API連携）

アイテム詳細シートモーダルはサーバーAPIと連携し、変更を永続化しなければならない（SHALL）。

- ステップの追加・完了トグル・削除はサーバーAPIを呼び出さなければならない（MUST）
- アイテムの編集・削除はサーバーAPIを呼び出さなければならない（MUST）
- 楽観的更新パターンでUIを即時更新しなければならない（SHALL）
- APIエラー時はユーザーにフィードバックを表示しなければならない（SHALL）

#### Scenario: ステップ完了トグル

- **WHEN** ユーザーがステップをタップする
- **THEN** UIが即座に更新される（楽観的更新）
- **AND** サーバーAPIが呼び出される
- **AND** エラー時はUIがロールバックされトースト通知が表示される

#### Scenario: ステップ追加

- **WHEN** ユーザーがステップ追加フォームからステップを追加する
- **THEN** サーバーAPIが呼び出される
- **AND** 作成されたステップがリストに追加される
- **AND** エラー時はトースト通知が表示される

#### Scenario: ステップ削除

- **WHEN** ユーザーがステップをスワイプして削除する
- **THEN** UIが即座に更新される（楽観的更新）
- **AND** サーバーAPIが呼び出される
- **AND** エラー時はUIがロールバックされトースト通知が表示される

#### Scenario: アイテム編集

- **WHEN** ユーザーが編集ボタンを押す
- **THEN** タイトル編集モードになる
- **AND** 保存時にサーバーAPIが呼び出される
- **AND** 成功時は親コンポーネントに更新が通知される

#### Scenario: アイテム削除

- **WHEN** ユーザーが削除ボタンを押す
- **THEN** 確認ダイアログが表示される
- **AND** 確認後にサーバーAPIが呼び出される
- **AND** 成功時はシートが閉じられ親コンポーネントに削除が通知される

## ADDED Requirements

### Requirement: AIステップ提案機能

アイテム詳細シートからAIによるステップ提案を利用できなければならない（SHALL）。

- 「もっとステップを提案」ボタンでAI提案APIを呼び出さなければならない（SHALL）
- 既存のステップと完了済みステップを踏まえた提案を取得しなければならない（SHALL）
- 提案されたステップから選択して追加できなければならない（SHALL）
- ローディング中はスケルトンUIを表示しなければならない（SHALL）

#### Scenario: AI提案の取得

- **WHEN** ユーザーが「もっとステップを提案」ボタンを押す
- **THEN** ローディングUIが表示される
- **AND** AI提案APIが呼び出される
- **AND** 既存ステップと完了済みステップが提案APIに送信される

#### Scenario: AI提案からステップ追加

- **WHEN** AI提案が表示される
- **AND** ユーザーが提案されたステップを選択する
- **AND** 追加ボタンを押す
- **THEN** 選択されたステップがサーバーAPIで作成される
- **AND** ステップリストに追加される

#### Scenario: AI提案APIエラー

- **WHEN** AI提案APIがエラーを返す
- **THEN** エラーメッセージが表示される
- **OR** デフォルトのフォールバック提案が表示される

## EDGE CASES

### Edge Case: 空のステップタイトル

- **WHEN** ユーザーが空のタイトルでステップを追加しようとする
- **THEN** 追加ボタンは無効化されている
- **AND** APIは呼び出されない

### Edge Case: ネットワークエラー中の操作

- **WHEN** ネットワーク接続がない状態で操作を行う
- **THEN** 楽観的更新でUIは即座に更新される
- **AND** APIエラー時にUIがロールバックされる
- **AND** エラートーストが表示される

### Edge Case: 連続した高速操作

- **WHEN** ユーザーが複数のステップを高速で連続トグルする
- **THEN** 各操作は独立してAPIに送信される
- **AND** 操作中のステップにはローディングインジケータが表示される
- **AND** 後続の操作が先行する操作の完了を待たない

### Edge Case: 削除中のアイテムへの操作

- **WHEN** アイテム削除APIが実行中
- **THEN** シート内の他の操作ボタンは無効化される
- **AND** 削除完了までローディング状態が表示される

### Edge Case: 長いタイトルの編集

- **WHEN** ユーザーが非常に長いタイトル（100文字以上）を入力する
- **THEN** 入力は制限されるか、適切にトランケートされる
- **AND** UIが崩れない
