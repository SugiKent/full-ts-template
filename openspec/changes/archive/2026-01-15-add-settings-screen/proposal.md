# Proposal: add-settings-screen

## What

モバイルアプリに設定画面を実装する。Home画面右上のアイコンをプロフィールアイコンに変更し、以下の機能を提供する：

- **テーマカラー設定** - 背景色・差し色・Primary/Secondary カラーのカスタマイズ
- 利用規約へのリンク
- プライバシーポリシーへのリンク
- ユーザー（デバイス）データの削除

## Why

1. **設定画面の不在**: 現在の設定画面は未実装状態（TODO）であり、ユーザーが法的文書を閲覧したり、データを削除する手段が提供されていない
2. **パーソナライズ需要**: ユーザーがアプリの見た目を自分好みにカスタマイズできることで、愛着とエンゲージメントが向上する
3. **アプリストア要件**: 法的文書へのリンクはアプリストア申請時の必須要素

### 詳細

#### 1. Home画面のヘッダー変更

- 右上の歯車アイコン（Settings）をプロフィールアイコン（User）に変更
- タップ時の遷移先は変わらず `/settings`

#### 2. 設定画面の実装

- **プロフィールセクション**: アイコン + 匿名ユーザー表記
- **テーマカラーセクション**: カラーパレット選択UI
- **法的文書セクション**: 利用規約・プライバシーポリシーへのリンク
- **データ管理セクション**: ユーザーデータ削除機能
- **アプリ情報セクション**: バージョン情報

#### 3. テーマカラーカスタマイズ

設定可能なカラー：

| 項目 | 説明 | デフォルト |
|------|------|-----------|
| **背景色（Background）** | 画面全体の背景色 | amber-50 |
| **プライマリカラー（Primary）** | アクセント、ボタン等 | amber-600 |
| **セカンダリカラー（Secondary）** | サブアクセント | stone-500 |
| **テキストカラー（Text）** | メインテキスト | stone-800 |

#### 4. ユーザーデータ削除フロー

- 確認アラートを表示（「本当に削除しますか？この操作は取り消せません」）
- 確認後、`deleteDeviceData` API を呼び出し
- 成功後、ローカルストレージをクリアしてオンボーディング画面にリダイレクト
