## Requirements
### Requirement: タイムライン統合表示

タイムライン画面はジャーナルエントリーと完了ログを時系列で統合表示しなければならない（SHALL）。

#### Scenario: タイムライン一覧の表示

- **WHEN** ユーザーがタイムライン画面を開く
- **THEN** ジャーナルエントリーと完了ログが逆時系列（新しい順）で混在表示される
- **AND** 日付ごとにグループ化されて表示される
- **AND** 日付ヘッダーには「今日」「昨日」または日付が表示される

#### Scenario: エントリータイプの視覚的区別

- **WHEN** タイムラインが表示される
- **THEN** ジャーナルエントリーは📝アイコンとともに表示される
- **AND** アイテム完了ログは🎉アイコンと「[アイテム名] を達成しました」形式で表示される
- **AND** ステップ完了ログは✅アイコンと「[ステップ名] を完了しました」形式で表示される

#### Scenario: タイムラインが空の場合

- **WHEN** ジャーナルエントリーも完了ログも存在しない
- **THEN** 「今日の気づきを書いてみましょう」のような案内メッセージが表示される
- **AND** ジャーナル入力への導線が表示される

### Requirement: ジャーナルエントリー作成

ユーザーは日々の気づきや振り返りをジャーナルエントリーとして作成できなければならない（SHALL）。

#### Scenario: 新規エントリーの作成

- **WHEN** ユーザーがタイムライン画面で入力ボタン（FAB）をタップする
- **THEN** ジャーナル入力用シートモーダルが表示される
- **WHEN** ユーザーが本文を入力して保存する
- **THEN** ジャーナルエントリーが作成される
- **AND** タイムラインに追加される

#### Scenario: タイトル付きエントリーの作成

- **WHEN** ユーザーがタイトルと本文を入力する
- **AND** 保存ボタンをタップする
- **THEN** タイトル付きのエントリーが作成される

#### Scenario: タイトルなしエントリーの作成

- **WHEN** ユーザーが本文のみを入力する
- **AND** 保存ボタンをタップする
- **THEN** タイトルなしのエントリーが作成される

#### Scenario: 本文の文字数制限

- **WHEN** ユーザーが5000文字を超える本文を入力しようとする
- **THEN** 5000文字以上は入力できない
- **AND** 残り文字数が表示される

### Requirement: ジャーナルエントリー編集

ユーザーは既存のジャーナルエントリーを編集できなければならない（SHALL）。

#### Scenario: エントリーの編集

- **WHEN** ユーザーがジャーナルエントリーをタップする
- **THEN** エントリーの詳細が表示される
- **AND** 編集ボタンが表示される
- **WHEN** ユーザーが編集して保存する
- **THEN** エントリーが更新される

### Requirement: ジャーナルエントリー削除

ユーザーはジャーナルエントリーを削除できなければならない（SHALL）。

#### Scenario: エントリーの削除

- **WHEN** ユーザーがジャーナルエントリーの削除操作を行う
- **THEN** 確認ダイアログが表示される
- **WHEN** ユーザーが削除を確定する
- **THEN** エントリーが削除される
- **AND** タイムラインから消える

#### Scenario: 削除のキャンセル

- **WHEN** ユーザーが確認ダイアログでキャンセルを選択する
- **THEN** エントリーは削除されない

### Requirement: 完了ログ表示

タイムラインにはウィッシュリストアイテムとステップの完了ログが自動的に表示されなければならない（SHALL）。

#### Scenario: アイテム完了ログの表示

- **WHEN** ウィッシュリストアイテムが完了している
- **THEN** 完了日時にアイテム完了ログがタイムラインに表示される
- **AND** アイテム名と完了日時が表示される

#### Scenario: ステップ完了ログの表示

- **WHEN** ステップが完了している
- **THEN** 完了日時にステップ完了ログがタイムラインに表示される
- **AND** ステップ名と完了日時が表示される

#### Scenario: 完了ログは読み取り専用

- **WHEN** ユーザーが完了ログをタップする
- **THEN** 関連するアイテム/ステップの詳細が表示される
- **AND** 完了ログ自体の編集・削除はできない

### Requirement: スワイプタブナビゲーション

タイムライン画面はHOME画面の左側にスワイプで移動可能なタブとして配置されなければならない（SHALL）。

#### Scenario: HOMEからタイムラインへの移動

- **WHEN** ユーザーがHOME画面で右から左にスワイプする
- **THEN** タイムライン画面が表示される

#### Scenario: タイムラインからHOMEへの移動

- **WHEN** ユーザーがタイムライン画面で左から右にスワイプする
- **THEN** HOME画面が表示される

#### Scenario: タブインジケーターの表示

- **WHEN** タブ画面が表示される
- **THEN** 現在のタブ位置を示すインジケーター（ドット）が表示される

### Requirement: HOMEからタイムラインへの導線

HOME画面からタイムライン画面への導線が存在しなければならない（SHALL）。

#### Scenario: ヘッダーからタイムラインへ移動

- **WHEN** ユーザーがHOME画面のヘッダー左側にあるタイムラインアイコンをタップする
- **THEN** タイムライン画面に切り替わる

### Requirement: タイムラインページネーション

タイムラインは大量のエントリーを効率的に表示するためページネーションを実装しなければならない（SHALL）。

#### Scenario: 初期表示

- **WHEN** タイムライン画面が開かれる
- **THEN** 最新50件のエントリーが表示される

#### Scenario: 追加読み込み

- **WHEN** ユーザーがタイムラインの最下部までスクロールする
- **THEN** 次の50件が追加で読み込まれる
- **AND** ローディングインジケーターが表示される

#### Scenario: Pull To Refresh

- **WHEN** ユーザーがタイムライン一覧を下方向に引っ張る
- **THEN** リフレッシュインジケーターが表示される
- **AND** 最新データでタイムラインが更新される

### Requirement: Notion風ミニマルデザイン

タイムライン画面はNotion風のミニマルデザインを採用しなければならない（SHALL）。

#### Scenario: デザインガイドライン適用

- **WHEN** タイムライン画面が表示される
- **THEN** 白・黒・グレーを基調としたカラーパレットが使用される
- **AND** 十分な余白が確保される
- **AND** カード型UIでエントリーが表示される
- **AND** アイコンは線画（アウトライン）スタイルで統一される

### Requirement: タイムラインAPI

タイムライン機能はサーバーAPIを通じてデータを永続化しなければならない（SHALL）。

#### Scenario: タイムライン取得API

- **WHEN** クライアントが `getTimeline` API を呼び出す
- **AND** 有効な認証トークンが提供される
- **THEN** ジャーナルエントリーと完了ログが統合されて返される
- **AND** 逆時系列でソートされている
- **AND** ページネーション用のカーソルが含まれる

#### Scenario: ジャーナル作成API

- **WHEN** クライアントが `createJournalEntry` API を呼び出す
- **AND** 有効な認証トークンが提供される
- **THEN** ジャーナルエントリーがデータベースに保存される
- **AND** 作成されたエントリーが返される

#### Scenario: ジャーナル更新API

- **WHEN** クライアントが `updateJournalEntry` API を呼び出す
- **AND** 有効な認証トークンが提供される
- **AND** 該当デバイスが所有するエントリーIDが指定される
- **THEN** エントリーが更新される

#### Scenario: ジャーナル削除API

- **WHEN** クライアントが `deleteJournalEntry` API を呼び出す
- **AND** 有効な認証トークンが提供される
- **AND** 該当デバイスが所有するエントリーIDが指定される
- **THEN** エントリーが削除される
