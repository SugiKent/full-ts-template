## Requirements
### Requirement: 設定画面のナビゲーション

Home画面右上にプロフィールアイコンを表示し、タップで設定画面に遷移できなければならない（SHALL）。

- アイコンは lucide-react-native の User アイコンを使用しなければならない（MUST）
- タップで `/settings` ルートに遷移しなければならない（SHALL）
- アイコンサイズは 22px、カラーは `#78716C`（stone-500）とする（SHALL）

#### Scenario: プロフィールアイコンの表示

- **WHEN** Home画面が表示される
- **THEN** 右上にプロフィールアイコン（User）が表示される
- **AND** アイコンは白背景の丸ボタン内に配置される

#### Scenario: 設定画面への遷移

- **WHEN** ユーザーがプロフィールアイコンをタップする
- **THEN** 設定画面に遷移する

### Requirement: 設定画面レイアウト

設定画面は複数のセクションで構成され、スクロール可能でなければならない（SHALL）。

- プロフィールセクション（匿名ユーザー情報）
- テーマセクション（カラーテーマ選択）
- 法的文書セクション（利用規約、プライバシーポリシー）
- データ管理セクション（データ削除）
- アプリ情報セクション（バージョン）

#### Scenario: 設定画面の初期表示

- **WHEN** ユーザーが設定画面を開く
- **THEN** 画面上部にプロフィールセクションが表示される
- **AND** 各セクションが縦に並んで表示される
- **AND** 画面はスクロール可能である

### Requirement: プロフィールセクション

設定画面上部にユーザープロフィール情報を表示しなければならない（SHALL）。

- アバターアイコン（デフォルトのユーザーアイコン）を表示しなければならない（SHALL）
- 「ゲストユーザー」または「匿名ユーザー」と表示しなければならない（SHALL）

#### Scenario: プロフィールの表示

- **WHEN** 設定画面が表示される
- **THEN** 画面上部にユーザーアイコンが表示される
- **AND** 「ゲストユーザー」というテキストが表示される

### Requirement: 法的文書セクション

利用規約とプライバシーポリシーへのリンクを提供しなければならない（SHALL）。

- 「利用規約」と「プライバシーポリシー」の2つのリンクを表示しなければならない（SHALL）
- 各リンクはタップ可能でなければならない（MUST）
- タップ時は外部ブラウザで該当URLを開かなければならない（SHALL）

#### Scenario: 利用規約へのアクセス

- **WHEN** ユーザーが「利用規約」をタップする
- **THEN** 外部ブラウザが起動する
- **AND** 利用規約ページ（`/legal/terms.html`）が表示される

#### Scenario: プライバシーポリシーへのアクセス

- **WHEN** ユーザーが「プライバシーポリシー」をタップする
- **THEN** 外部ブラウザが起動する
- **AND** プライバシーポリシーページ（`/legal/privacy.html`）が表示される

### Requirement: データ削除機能

ユーザーが自身のすべてのデータを削除できなければならない（SHALL）。

- 「データを削除」ボタンを表示しなければならない（SHALL）
- 削除は確認ダイアログを経由しなければならない（MUST）
- 削除成功後はオンボーディング画面にリダイレクトしなければならない（SHALL）

#### Scenario: データ削除の確認

- **WHEN** ユーザーが「データを削除」ボタンをタップする
- **THEN** 確認アラートが表示される
- **AND** 「本当に削除しますか？この操作は取り消せません」というメッセージが表示される
- **AND** 「削除」と「キャンセル」のボタンが表示される

#### Scenario: データ削除の実行

- **WHEN** ユーザーが確認アラートで「削除」をタップする
- **THEN** `deleteDeviceData` API が呼び出される
- **AND** ローカルストレージ（SecureStore, AsyncStorage）がクリアされる
- **AND** オンボーディング画面にリダイレクトされる

#### Scenario: データ削除のキャンセル

- **WHEN** ユーザーが確認アラートで「キャンセル」をタップする
- **THEN** アラートが閉じる
- **AND** データは削除されない
- **AND** 設定画面に留まる

### Requirement: アプリ情報セクション

アプリのバージョン情報を表示しなければならない（SHALL）。

- アプリ名を表示しなければならない（SHALL）
- バージョン番号を表示しなければならない（SHALL）

#### Scenario: バージョン情報の表示

- **WHEN** 設定画面が表示される
- **THEN** アプリ名「My Wishlist」が表示される
- **AND** 現在のバージョン番号が表示される

### Requirement: 設定画面のデザイン

設定画面は全体のデザインガイドラインに準拠しなければならない（SHALL）。

- 現在選択中のテーマカラーを適用しなければならない（MUST）
- セクション間には適切な余白を設けなければならない（SHALL）
- タップ可能な要素は視覚的にわかりやすくしなければならない（SHALL）

#### Scenario: テーマに応じた表示

- **WHEN** ユーザーがテーマを変更する
- **THEN** 設定画面のカラーが即座に更新される
